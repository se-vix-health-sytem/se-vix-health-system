Detailed Description

Manages hospital resource inventory — stock levels across all storage facilities, low-stock detection, and resource intake/consumption flows.

Covers FR5.5 (inventory visibility) and UC25 (employee resource consumption) and UC27 (buyer restocking).

Annotated @Transactional(readOnly=true) at the class level; write methods override with @Transactional.

Two actor-aware overloads exist for stock mutations: the domain-method path (accepts an Employee or Buyer actor) enforces business rules inside the model, while the ID-only overloads are provided for internal/system use when no actor is in scope.
