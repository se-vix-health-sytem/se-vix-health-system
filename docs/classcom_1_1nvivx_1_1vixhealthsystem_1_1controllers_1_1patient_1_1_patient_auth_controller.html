Detailed Description

Controller for patient authentication, dashboard, profile, and account management — base URL /patient.

Patients authenticate using their Italian fiscal code (codice fiscale) either directly through the portal form or via the mock SPID/CIE government-identity flow in MockSpidController. On successful authentication a patient object is stored in the HTTP session and a Spring Security context is registered so that route-level access rules work correctly. Every significant action (login, logout, deletion) is written to the audit log.

POST /patient/authenticate — log in a patient using their fiscal code.

Looks up the patient by fiscal code, stores the entity in the HTTP session under the "patient" key, registers a Spring Security context so that route guards work, and audits the login event. Redirects to an optional custom URL or to the dashboard by default.

String deleteAccount ( @RequestParam(required=false) String confirm,
HttpSession session,
Model model )

POST /patient/delete — anonymize and delete the patient's account.

Requires the patient to submit the string "CONFIRM" as the confirm parameter; any other value (or an absent value) renders a confirmation page instead. Medical records are retained for five years per GDPR Article 17(3)(c) even though personal identifiers are anonymized.

Parameters
confirmMust equal "CONFIRM" (case-sensitive) to proceed.
sessionHTTP session; invalidated on successful deletion.
modelReceives requireConfirm=true when confirmation is needed, or a message attribute on success.
Returns
Redirect to / on success, the patient/confirm-delete template when confirmation is pending, or redirect:/patient/login when no session patient is found.
