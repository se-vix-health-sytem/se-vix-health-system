Detailed Description

Shared enumerations used across the domain model.

Using enums instead of raw strings gives us compile-time safety and makes it impossible to accidentally store an invalid status value in the database.

  • AppointmentStatus — PENDING, CONFIRMED, CANCELLED, RESCHEDULED
  • BedStatus — FREE / OCCUPIED (computed, not stored)
  • EmployeeType — discriminator values for the employee hierarchy
  • MachineStatus — operational state of medical machinery
  • PaymentStatus — PAID / UNPAID for appointment billing
  • Role — Spring Security authority names
  • ShiftType — MORNING, AFTERNOON, NIGHT
  • VacationStatus — PENDING, APPROVED, DENIED

Backward-compatible getters (returning String) are provided on the entities that use these enums so that Thymeleaf templates and Jackson serialisation don't break on existing data.
