Detailed Description

Authentication controller for staff (employees) of VIX Health System.

Handles login, Firebase-backed credential verification, role-based dashboard redirection, and logout for all employee roles (MedicalSpecialist, Secretary, Technician, Buyer, StaffManager). Also exposes a dev-only role-selector endpoint that bypasses Firebase for local testing.

Base URL: no prefix (login/logout live at the root context).

Use cases covered:

  • UC01 Staff login via Firebase email/password
  • Dev quick-login via pre-seeded credential store

POST /authenticate — verify Firebase credentials and open a staff session.

Calls Firebase to exchange email/password for a UID, looks up the matching Employee, registers a Spring Security context in the HTTP session, then redirects to the role-specific dashboard. Returns to the login page with an error attribute on any failure.

The session attribute "user" stores the Employee entity instead of using the Spring Security principal because employee domain objects carry additional context (department, specialty, etc.) that the principal does not.

Parameters
usernameEmployee's Firebase email address.
passwordFirebase account password (plain text over HTTPS).
sessionHTTP session; receives "user" and "role" attributes.
requestRaw servlet request (passed to session-security plumbing).
responseRaw servlet response (passed to session-security plumbing).
modelReceives "error" when authentication fails.
Returns
Redirect to the role-specific dashboard, or the login template with an error message.
