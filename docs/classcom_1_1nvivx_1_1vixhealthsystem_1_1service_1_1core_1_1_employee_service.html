Detailed Description

Central service for employee lifecycle management — creation, lookup, update, deletion, and Firebase account synchronisation.

Annotated @Transactional(readOnly=true) at the class level so that all read operations benefit from connection-pool optimisations. Individual write operations override with @Transactional.

Every mutating action is recorded via AuditService to satisfy NFR02 (traceability). Firebase accounts are provisioned alongside the database record so that employees can log in immediately after creation.

void deleteEmployee ( Long id)

Deletes an employee from both the database and Firebase Authentication.

The Firebase account is removed first; if that fails the database record is not deleted to avoid orphaned auth entries. The dev credential entry is cleaned up regardless of Firebase success to keep the demo board tidy.
