Detailed Description

Configures Spring Security for role-based URL access control.

Authentication is handled entirely by Firebase (see FirebaseConfig). Spring Security's job here is authorisation only — it checks the role stored in the HTTP session and either lets the request through or redirects to the appropriate login page.

CSRF, form-login, and HTTP-Basic are all disabled because the app uses Firebase ID tokens and session attributes for auth state rather than Spring's built-in mechanisms.

SecurityFilterChain filterChain ( HttpSecurity http) throws Exception

Declares the security filter chain that governs every HTTP request.

Access rules summary:

  • Public pages (site, assets, login endpoints) — permitAll()
  • /patient/** — ROLE_PATIENT only
  • /medical-specialist/** — ROLE_MEDICALSPECIALIST only
  • /secretary/** — ROLE_SECRETARY only
  • /buyer/** — ROLE_BUYER only
  • /technician/** — ROLE_TECHNICIAN only
  • /staff-manager/** — ROLE_STAFFMANAGER only
  • /payment/** — patient or secretary
  • /employee/resources/** — any staff role

On auth failure: patient paths redirect to /patient/login, all others to /login.

Parameters
