Detailed Description

Manages appointment booking, rescheduling, and cancellation for the VIX Health System.

Appointments are stored in a JSON file via JsonAppointmentRepository rather than a relational table, so there is no @Transactional support; all reads and writes are synchronized by the repository implementation.

Slot-conflict detection is enforced before every new booking to prevent double-booking across the shared appointment list.

Books an appointment on behalf of a patient through a secretary (UC-Secretary flow).

Delegates to Secretary#makeAppointmentForPatient so that domain invariants (e.g., patient eligibility) are enforced by the model. The slot-conflict guard runs before the domain call to fail fast.
