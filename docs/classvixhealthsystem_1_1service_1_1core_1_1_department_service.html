Detailed Description

Manages hospital departments and their associated specialists.

Covers department listing, doctor lookup, and department-service catalogue queries.

Not annotated @Transactional(readOnly=true) at the class level because most operations are already read-only and the service performs no writes; individual write paths can opt in if added in the future.

Map< Long, String > getAllDoctorImageMap ( )

Builds a stable specialistId → image-path map for every specialist across all departments using only two database queries (all departments + all employees).

Male and female counters reset per department; the image pool wraps at index 2 to avoid null paths when there are more than two doctors of the same gender.

Returns
LinkedHashMap ordered by department iteration order; never null.

Map< Long, String > getDoctorImageMap ( Long departmentId)

Builds a stable specialistId → image-path map for a single department.

Doctors are sorted by ID so that adding or removing staff never reshuffles existing photo assignments. The counter wraps at 2 per gender to stay within the available static asset set.
