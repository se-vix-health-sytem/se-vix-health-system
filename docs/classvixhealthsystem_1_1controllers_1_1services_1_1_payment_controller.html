Detailed Description

Controller for the payment workflow — base URL /payment.

Manages the end-to-end payment flow for appointment fees: showing the payment form, processing a real card payment via PaymentService, querying transaction status, and a fake-pay shortcut for demo/test purposes that simulates a successful transaction without real card data. After a successful payment the appointment status is set to CONFIRMED.

POST /payment/fake-pay — simulate a successful payment for demo purposes.

Builds a hard-coded PaymentRequest with a test card number and calls the real payment service path so that the same confirmation and status-update logic runs as in a live transaction. On success the appointment is marked as paid and CONFIRMED before redirecting.

Parameters
