Detailed Description

Manages employee shift assignments, backed by a JSON file rather than the database.

Shifts are stored in src/main/resources/storage/shifts.json. The JSON approach was chosen because shift data changes frequently and has no relational dependencies that require foreign-key constraints. Every write is flushed to disk immediately (no caching) so the file always reflects the current state.

The ID sequence is rebuilt from the existing JSON on startup, which means IDs continue incrementing even after a server restart — they are never reused.

All mutating operations are audit-logged via AuditService.
