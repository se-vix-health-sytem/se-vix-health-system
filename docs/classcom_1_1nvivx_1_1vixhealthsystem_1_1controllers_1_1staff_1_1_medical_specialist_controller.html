Detailed Description

Controller for MedicalSpecialist staff members — base URL /medical-specialist.

Medical specialists use this controller to manage clinical activities: searching patient records, adding diagnoses, prescriptions, exam results, and scheduling surgeries. Additional views cover the specialist's own appointments, a monthly calendar, a day-by-day schedule, and their profile.

Only accessible to users with ROLE_MEDICALSPECIALIST.

Use cases covered: UC10 (view medical records), UC11 (add diagnosis), UC12 (add prescription), UC13 (schedule surgery), UC18 (appointments).

POST /medical-specialist/patients/{patientId}/add-surgery — schedule a surgery for a patient.

The acting specialist is resolved from the session to invoke the domain method MedicalSpecialist.scheduleSurgeryForPatient. The session is consulted rather than the Spring Security principal because only the full MedicalSpecialist domain object carries the scheduling business logic. The operation is audited via AuditService.

Parameters
patientIdDatabase ID of the patient receiving the surgery.
surgeryNameName / type of the surgery.
surgeryDescriptionOptional narrative description; defaults to empty string.
surgeryDateTimeISO datetime string from an HTML datetime-local input (yyyy-MM-ddTHH:mm or yyyy-MM-ddTHH:mm:ss).
roomIdOptional ID of the SpecializedRoom to use.
sessionHTTP session carrying the "user" Employee attribute.
redirectAttributesFlash attributes for the redirect.
Returns
Redirect to /medical-specialist/patients/{patientId}/record.
