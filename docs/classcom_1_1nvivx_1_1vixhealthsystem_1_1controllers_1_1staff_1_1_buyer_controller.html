Detailed Description

Controller for Buyer staff members — base URL /buyer.

Buyers are responsible for procurement and inventory management. This controller covers the full purchasing workflow: viewing and updating stock levels, adding new resources, monitoring low-stock alerts, reviewing the resource-take audit log, and accessing personal shift and profile pages.

Only accessible to users with ROLE_BUYER.

Use cases covered: UC25 (resource management), UC30 (profile/shifts).

GET /buyer/inventory — list all resources currently in storage.

Aggregates stock quantities across all storage locations and wraps each entry in an InventoryItem for the view. Resources with fewer than 50 units are flagged as low-stock. An empty list is substituted and an error banner is shown if the service call fails.

Parameters
modelReceives resources (sorted by name), pageTitle, and isLowStockView attributes.

POST /buyer/inventory/add — create a new resource and add initial stock.

Creates the resource entity via InventoryService, then calls the domain method Buyer.addResourceToStorage to record the initial quantity. The acting buyer is resolved from the session.

Parameters

descriptionShort description of the resource.
