Detailed Description

Custom exceptions and the global error handler.

Every exception here maps to a specific business condition — they exist so that service code can throw something meaningful rather than a generic RuntimeException, and so that GlobalExceptionHandler can map each one to the right HTTP response or error page.

Domain exceptions:

  • AppointmentConflictException — time slot is already taken
  • SlotNotAvailableException — requested slot doesn't exist or is closed
  • PatientNotFoundException — patient ID not found in the database
  • ResourceNotFoundException — inventory resource not found
  • VacationNotFoundException — vacation request ID doesn't exist
  • UnauthorizedAccessException — caller lacks permission for the operation
